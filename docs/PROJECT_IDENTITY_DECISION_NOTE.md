## Decision

The repository keeps exact GR as its benchmark and public scientific identity.

At current scope, `The Æther-Flow Interpretation of Relativity` is:

- an exact-GR benchmark and interpretive package at observable scale
- plus an open foundational derivation program that is now restricted to genuinely new lines

At current scope, it is not:

- a completed first-principles derivation of GR
- a low-energy modification of GR
- a low-energy replacement theory of GR

## Why Exact GR Remains The Correct Benchmark

Exact GR remains the correct benchmark because it is the strongest positive result already in hand and the only observer-level law the active benchmark package presently controls cleanly.

More concretely:

- the overview-first exact-closure sequence already fixes one operative metric, universal matter coupling, and exact Einsteinian observer-level dynamics
- the benchmark package already recovers null structure, proper time, redshift, and weak-field phenomenology at GR level
- the active ontology is presently disciplined as interpretation of that exact relativistic package rather than as a separate low-energy field content
- the frozen primitive-reservoir line did not derive a different low-energy law; it ended instead in `Not Derived On Current Line`

The correct public identity is therefore the benchmark package actually secured, not a stronger low-energy gravity claim that the repository has not earned.

## Why Low-Energy Modification Or Replacement Is Not The Current Goal

Low-energy GR modification or replacement is not the current goal because that would change the scientific burden of the project rather than simply extending the present benchmark package.

At current scope, the repository does not have:

- a supported second operative metric
- a supported extra observer-level gravity field sector
- a supported independent low-energy non-GR observable signature
- a settled observational or theoretical case for replacing the exact-GR benchmark already in hand

Changing the flagship project now would blur three different questions that need to stay separate:

1. what exact relativistic package is already established
2. whether a genuinely new substrate line can derive that package
3. whether some future theory should instead deviate from GR at low energy

The active repository should keep questions (1) and (2) together and defer question (3) unless a different project is intentionally opened.

## Conditions For A Separate Non-GR Project Later

A separate GR-modification or GR-replacement project may be opened later only if all of the following are true:

- the exact-GR benchmark package is intentionally abandoned or demoted as the flagship identity
- the new project states explicit low-energy deviations rather than vague dissatisfaction with GR
- the new project identifies concrete observable consequences, regime changes, or new propagating sectors
- the new project accepts the burden of current observational and theoretical constraints
- the new project is renamed and re-scoped explicitly as a different project with a different claim boundary

If those conditions are not met, the repository should not describe itself as moving beyond exact GR.

## Operational Consequence

For current repository work, the decision is:

- keep the exact-GR benchmark package as Product A and as the public front door
- keep the frozen bounded no-go as Product B and as downstream status, not as replacement identity
- keep any resumed derivational work subordinate to that benchmark and gated by [NEW_LINE_DERIVATION_CHARTER.md](NEW_LINE_DERIVATION_CHARTER.md)
- do not present a new low-energy gravity theory unless a separate project has been opened explicitly

## Conclusion

The project identity is now explicit:

- current flagship identity: exact-GR benchmark interpretation
- current open burden: whether a genuinely new line can derive that benchmark
- deferred possibility: a separate future non-GR project, only under an explicitly different scope

This note is intended to remove ambiguity, not to foreclose later creativity.
It keeps the present repository honest about what has already been established, what remains open, and what would require a different project altogether.
