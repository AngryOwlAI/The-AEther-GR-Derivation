# GR Derivation Reset Plan

Updated: April 13, 2026

Status: Proposed reset only. This document does not implement the reset. It defines the plan by which the repository could be reset from open-ended recursive frontier motion to a short, bounded GR-derivation program.

## Purpose

This plan converts the current audit result into a concrete repository reset.

The reset has one aim:

- force the active derivational line to converge to a bounded answer about GR rather than continue indefinitely through deeper same-output relay manuscripts

This plan does **not** change the benchmark claim boundary:

- the exact-closure package remains the public front door
- GR remains adopted exactly at effective scope in that package
- the derivational line remains downstream of that benchmark

## Audit-Based Decision

The audit supports four conclusions.

1. The repository already has a finished exact-relativistic benchmark package by adoption.
2. The repository does **not** yet have a completed first-principles derivation of GR from explicit substrate variables.
3. The repository does **not** yet have a general impossibility theorem proving that such a derivation cannot be achieved on the active line.
4. The active derivational line has fallen into recursive depth drift: repeated promotion of deeper same-output relay manuscripts whose main effect is to move the unexplained layer downward.

The reset therefore treats the current situation as:

- benchmark theory in hand
- derivation still open
- recursion no longer acceptable as the default research mode

## Reset Principle

After adoption of this plan, the active derivational program is no longer allowed to justify itself by "going one layer deeper" alone.

A main-line manuscript will count as legitimate progress only if it moves the project toward one of three bounded outcomes:

1. a completed derivation of GR from explicit substrate structure
2. a stable obstruction or no-go result showing that the current line cannot deliver that derivation under its present assumptions
3. a suspension verdict stating that the line has reached the limit of what can be responsibly claimed without a qualitatively new substrate law

There is no fourth allowed outcome in which the project continues indefinitely by renaming the deepest remaining package.

## Hard Stop-Rule

The following rule is the center of the reset.

### Main-line admission rule

A new manuscript may be promoted as main-line progress only if it is one of the following:

- a bounded derivation attempt manuscript
- a bounded obstruction or no-go manuscript
- a bridge-object compression or necessity manuscript that directly shortens the bounded derivation attempt and does not merely move the unexplained layer deeper

### Automatic non-promotion rule

A manuscript must **not** be promoted to `active_primary` if it does all of the following:

- preserves the same one operative metric
- preserves the same observer-equation closure verdict
- preserves the same benchmark-facing output
- leaves the exact benchmark package unchanged
- relocates the unexplained substrate layer to a deeper named package without discharging a benchmark derivation gate or producing a stable obstruction

This is the reset version of the existing safeguard against recursive depth drift.

### Forbidden default pattern

The following pattern is no longer admissible as the default next move:

- theorem: current package follows canonically from a smaller deeper package
- criterion: next burden therefore sits below that deeper package
- new theorem: next package follows canonically from a still smaller package
- repeat

### Criterion-file freeze

No new `*_next_benchmark_criterion.tex` file should be created unless it is attached to:

- a bounded derivation attempt with an explicit gate-by-gate proof burden
- or a bounded obstruction manuscript with an explicit failure claim

Ordinary routing handoff alone no longer justifies a new criterion file.

## Stable Bridge Object Under Test

For reset purposes, the derivational line must stop chasing the latest frontier label and instead freeze one benchmark-stable bridge object under test.

The bridge object to test is:

- the explicit substrate law that forces the primitive observer-reduction chain yielding the continuity-Hessian compressed core, the one operative metric, the ordinary matter route, and the benchmark one-metric observer package as derivational output rather than preserved package data

In the audit language, the best current repository-level candidate for that bridge object is:

- the primitive reversible constrained substrate dynamics normal form that forces the explicit equation / symmetry / constraint package and hence the primitive observer-reduction chain

The reset does **not** claim that this object is already sufficient.

It does claim that:

- this object is benchmark-stable enough to test directly
- the repo should stop replacing it by still deeper package names unless a bounded derivation attempt or bounded obstruction requires that replacement explicitly

### Program Step 3: Write One Bridge-Object Synthesis Manuscript

The next planned main-line manuscript should not be another frontier theorem.

It should be one synthesis manuscript that:

- identifies the stable bridge object under test
- states exactly which assumptions about substrate variables and maps remain primitive
- states exactly which benchmark-facing gains are already in hand
- states exactly which of the five derivation gates remain unproved from explicit substrate structure
- removes all "go deeper again" language from its conclusion

Recommended future filename:

- `manuscripts/active/tex/aether_flow_bounded_gr_derivation_bridge_object_synthesis.tex`

Acceptance condition:

- a reader should be able to answer "what still has to be shown?" without consulting a relay chain of later frontier files

### Program Step 4: Write One Bounded Derivation Attempt

After the synthesis manuscript, the next and only default main-line move is one bounded derivation attempt.

That manuscript must include all of the following explicitly:

- explicit substrate variables
- explicit substrate law or action, or an explicit theorem-level substitute if no action exists
- explicit map to the observer metric/matter sector
- explicit gate-by-gate derivation table for the five benchmark derivation gates
- exact statement of where the argument succeeds
- exact statement of where the argument fails, if it fails

Recommended future filename:

- `manuscripts/active/tex/aether_flow_bounded_gr_derivation_attempt.tex`

Acceptance condition:

- the conclusion must end in one of only three forms:
  - all five gates discharged
  - one or more gates not discharged for identified reasons
  - derivation undecidable on current substrate law because a specified object is still undefined

The conclusion may not end by naming a deeper package as the new honest burden unless that deeper package is introduced inside the same bounded attempt and tied to a specific gate failure.

### Program Step 5: If The Attempt Fails, Write One Obstruction Manuscript

If the bounded derivation attempt does not discharge the five gates, the next default move is not another deeper relay theorem.

It is one obstruction manuscript that proves one of the following:

- a local obstruction: the current active line cannot close a specified gate under its present assumptions
- a structural obstruction: the current one-metric primitive-reservoir line cannot derive the benchmark package without a new assumption, new variable, or new symmetry principle
- a scoped no-go: the current line can reproduce adoption plus preservation, but not derivation, at the required benchmark strength

Recommended future filename:

- `manuscripts/active/tex/aether_flow_current_line_gr_derivation_obstruction_or_no_go.tex`

Acceptance condition:

- the conclusion must identify which gate fails and whether the failure is branch-local, package-local, or line-level

### Program Step 6: Stop With A Verdict

After the bounded derivation attempt and, if needed, the obstruction manuscript, the repo must stop with one verdict.

Allowed verdicts:

- `Derived`: GR is derived from explicit substrate structure under the repo’s own five-gate standard.
- `Not Derived On Current Line`: the current line does not derive GR under its present assumptions.
- `Suspended Pending New Law`: the exact-closure package remains the active theory statement, but further derivational work is paused until a qualitatively new substrate law is supplied.

Not allowed:

- indefinite continuation through deeper same-output frontier replacement

## Planned Routing Changes

This section states the intended routing consequences once the reset is adopted. It does not perform them here.

### What stays front door

The exact-closure flagship package stays `front_door`.

### What stays active supporting

The following should remain active supporting or routing control:

- derivation-gates note
- post-bridge gate-status note
- upstream compression theorem
- the strongest benchmark-facing bridge-entry gains already on record

### What loses main-line promotion priority

The recursive relay chain below the stable bridge object should no longer drive the main line by default.

In practical routing terms:

- the current `active_primary` frontier should be treated as the current exploratory endpoint of the old relay method, not as proof that the method should continue
- later same-output relay manuscripts should be treated as `historical_but_kept_active` unless they enter directly into the bounded derivation attempt or bounded obstruction argument
- `*_next_benchmark_criterion.tex` files should default to historical routing records rather than live frontier markers

### What may hold `active_primary` after reset

After adoption of the reset, `active_primary` should be reserved for exactly one of:

- the bounded bridge-object synthesis manuscript
- the bounded derivation attempt manuscript
- the obstruction/no-go manuscript

The active frontier should no longer be a theorem whose main conclusion is that the burden moved one package deeper again.

## Planned Agent Rules

After adoption of this reset, agents should follow the rules below before drafting a new main-line manuscript.

### Retrieval rule

Before proposing a new main-line manuscript, the agent should first use the Manuscript Wiki workflow and the active manuscript file map to confirm:

- what the current benchmark package is
- what the live bridge object under test is
- whether the candidate manuscript is truly benchmark-facing or only another same-output relay

The wiki is the default retrieval layer for this reset, but not a replacement for the active `.tex` source.

### Four admission questions

An agent must answer `yes` to at least one of these before treating a draft as main-line work:

1. Does this manuscript discharge one of the five benchmark derivation gates from explicit substrate structure?
2. Does this manuscript prove that the current line cannot discharge one of those gates under its present assumptions?
3. Does this manuscript directly shorten the bounded derivation attempt by eliminating an explicit primitive assumption rather than merely renaming a deeper layer?
4. Does this manuscript force a project-level verdict within the bounded program?

If the answer is `no` to all four, the manuscript is not main-line progress.

### Conclusion test

If a manuscript conclusion can be summarized as:

- "the same benchmark package is preserved"
- "no stronger promotion language is licensed"
- "the remaining burden moves one level deeper again"

then it fails the reset admission test unless it is explicitly classified as historical support.

### No silent frontier promotion

No draft should be promoted silently from exploratory support to benchmark-facing frontier status.

Promotion requires an explicit statement of:

- which benchmark gate changed
- or which obstruction was proved
- or which primitive assumption in the bounded attempt was removed

## Planned File and Program Compression

The reset aims to replace a large open-ended theorem/criterion chain with a small core program.

Target active derivational core after reset:

- benchmark package
- derivation-gates note
- post-bridge gate-status note
- one bridge-object synthesis manuscript
- one bounded derivation attempt manuscript
- one obstruction/no-go manuscript, only if needed

Everything else remains preserved, citable, and searchable, but no longer defines the default next move.

## Planned Success Conditions

The reset is successful only if all of the following become true.

1. A reviewer can name the stable bridge object under test without using the latest frontier filename.
2. The repo has one bounded derivation attempt rather than many open-ended frontier theorems.
3. Every active main-line manuscript ends in a yes/no/conditional verdict rather than another recursive burden shift.
4. The file map no longer treats depth alone as a reason for `active_primary` status.
5. The derivational program becomes short enough that a future audit can answer the GR question without tracing a long frontier lineage.

## Planned Failure Conditions

The reset should be judged unsuccessful if any of the following continues.

- new theorem/criterion pairs whose main result is that the burden moved one layer deeper
- new `active_primary` promotion for a manuscript that preserves the same benchmark-facing object and only renames a deeper package
- conclusions that explicitly defer the answer again without either discharging a gate or proving an obstruction
- continued dependence on the latest frontier label to identify the supposed missing bridge object

## Final Decision Rule

Once this reset is adopted, the repository should use the following rule.

If the bounded derivation attempt cannot discharge all five benchmark derivation gates, and if the obstruction manuscript shows why the current line fails, then the correct scientific conclusion is:

- the current repository contains an exact-GR interpretive theory and an explored but unsuccessful derivation line, not a completed derivation of GR from first principles

If the bounded derivation attempt succeeds, then the correct scientific conclusion is:

- the current line has derived GR under the repo’s own benchmark standard

If neither conclusion can be reached because the substrate law is still too indeterminate, then the correct repository state is:

- exact-closure benchmark retained
- derivational line paused
- no further relay promotion until a qualitatively new substrate law is introduced

That is the stop-rule this reset is designed to impose.
