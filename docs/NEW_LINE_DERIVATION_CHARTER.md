## Purpose

This charter exists to block recursive depth drift.

No resumed derivational line may be opened, promoted, or summarized as live benchmark-facing progress unless this charter can be completed concretely first.

If the charter cannot be written concretely, do not open the line.
Outside review, reviewer encouragement, or packet return does not satisfy this gate by itself.

## Fixed Starting Point

Before filling this charter, keep the following repository facts fixed:

- the benchmark package is the overview-first exact-GR package already in hand
- the benchmark claim boundary remains one operative metric with ordinary matter coupling
- the frozen primitive-reservoir line already ends in `Not Derived On Current Line`
- another same-output deeper relay below the same frozen bridge object is not an admissible restart
- a resumed line must preserve the benchmark claim boundary unless the project is intentionally re-scoped as a separate non-GR project

## When This Charter Is Mandatory

Complete this charter before any of the following:

- proposing a new active derivational manuscript as front-line work
- promoting a candidate line to live repository priority
- describing a resumed derivational line as the next honest benchmark-facing move
- claiming that a new idea leaves the current obstruction / no-go behind

## Mandatory Pre-Screen Questions

The candidate line may proceed only if all of the following can be answered concretely.

1. What is the genuinely new localizing structure?
2. What exact mathematical object maps substrate data to observer spacetime fields?
3. How is the operative metric generated rather than merely inherited or preserved?
4. What prevents extra unsuppressed low-energy observer sectors?
5. Why is the proposal not just another same-output relay preserving the same benchmark-facing compressed core?

If any answer is missing, vague, or only verbal, the immediate recommendation is `No-Go`.

## Charter Fields

The charter for any resumed line must include all fields below.

### Minimum Mathematical Instantiation

Before a candidate line may be opened, promoted, or summarized as live benchmark-facing work, the charter must be instantiated at equation level rather than left as prose or schematic aspiration.

Required minimum form:

- an explicit local observer map with domain and codomain, for example `Phi_U : X|_U -> (g, psi, clock/null structure)|_U`
- an explicit operative-metric generation rule, for example `g_{mu nu} = F_{mu nu}(X, partial X, ldots)`
- an explicit observer-dynamics rule, for example `E^{obs}_{mu nu}[g, psi] = 0`, or an induced-action statement from which Einsteinian observer dynamics follow
- an explicit low-energy mode-control mechanism; it is not enough to promise that extra modes will later be projected out, screened, or tuned away
- an explicit same-metric recovery statement for null cones, proper time, and redshift from the generated operative metric
- an explicit verdict template stating what concrete evidence would count as `Derived`, `Not Derived On Current Line`, and `Suspended Pending New Law`

Automatic `No-Go` if:

- the charter remains verbal or metaphorical without equation-level local rules
- the local observer map, metric-generation rule, or observer-dynamics rule is deferred to later work
- mode control is asserted only as expectation rather than as a stated mechanism

### 1. New Ingredient

State the new ingredient explicitly.

Required content:

- name the ingredient
- classify it as an observer-localizing law, new primitive variable, or new symmetry principle
- write its mathematical form or governing rule
- explain why it is genuinely absent from the frozen line
- explain why it is not a relabeling of existing bridge-object data

Automatic `No-Go` if:

- the ingredient is only motivational language
- the ingredient is introduced only after the observer metric is already assumed
- the ingredient preserves the same frozen bridge object without changing the benchmark-facing burden

### 2. Local Observer Map

State how substrate data generate local observer spacetime fields.

Required content:

- define the substrate variables being used
- define the mathematical map from local substrate data to local observer fields
- state what counts as locality in the proposal
- say which observer fields are produced and on what domain

Preferred minimum form:

- a map such as `local substrate data -> observer fields`
- or, more explicitly, `Phi_U : X|_U -> (g, matter readout, clock/null structure)|_U`

Automatic `No-Go` if:

- observer fields appear only by interpretive declaration
- locality is left metaphorical
- the map is only global, averaged, or post hoc without a local field rule

### 3. Operative Metric Generation

State how the single operative metric arises.

Required content:

- identify the generating object or rule
- say whether the metric is derived, induced, or posited
- if induced, write the induction rule
- if posited, mark the line honestly as ansatz rather than derivation

Automatic `Not Derived On Current Line` if:

- an operative metric is simply inserted by hand and then treated as generated output
- a preserved benchmark metric is reused as if that already solved the derivation problem

### 4. Observer Dynamics

State how Einsteinian observer dynamics arise from the same local structure.

Required content:

- identify the derived observer equation
- explain how curvature dynamics arise from the substrate-side law
- say whether an Einstein-Hilbert term is derived, induced, or inserted
- if a standard gravitational action is inserted by hand, classify the line honestly as ansatz support only

Automatic `Not Derived On Current Line` if:

- Einsteinian dynamics are imported without a derivational account
- the proposal derives kinematics only while leaving dynamics external

### 5. Gauge / Mode Control

State why only GR-compatible low-energy observer modes remain.

Required content:

- identify the surviving low-energy sector
- audit extra scalar, vector, tensor, or hidden observer-side sectors
- explain the suppression, projection, symmetry, or gap mechanism
- state what prevents an enlarged low-energy matter or gravity law

Automatic `No-Go` if:

- extra observer-level sectors remain unsuppressed without being treated as a benchmark violation
- the proposal changes the one-metric benchmark claim boundary without explicitly opening a separate project

### 6. Relativistic Recovery

State how null cones, proper time, and redshift arise from the same metric.

Required content:

- identify the metric from which null structure is read
- explain how proper time is defined for matter observers
- explain how redshift and causal propagation follow from the same operative metric
- keep the recovery tied to the local observer map rather than to a separate interpretive layer

Automatic `Not Derived On Current Line` if:

- null cones, clocks, or redshift are borrowed from the adopted benchmark without generation from the candidate line
- different observer readouts require different operative metrics

### 7. Remainder Audit

State what prevents a genuine observer-side remainder.

Required content:

- identify any leftover observer-facing discrepancy
- classify it as absent, gauge, screened, high-energy only, or fatal
- explain why no benchmark-relevant remainder survives at observable scale
- say how this interacts with the benchmark derivation gates

Automatic `Not Derived On Current Line` if:

- a live observer-side remainder remains at benchmark scale
- the remainder is postponed to future work without verdict language

### 8. Failure Conditions And Verdict Rule

State in advance what result counts as each admissible outcome.

Required content:

- `Derived`
- `Not Derived On Current Line`
- `Suspended Pending New Law`

For this charter, `current line` means the candidate line presently under test.

Each verdict must say what specific evidence would trigger it.
A line is not allowed to remain open indefinitely just because more depth might exist.

## Additional Technical Rules

A candidate line must not call itself a derivation while relying on any of the following:

- direct insertion of an Einstein-Hilbert term without a derivational account
- use of a preserved observer metric as if that were generated output
- exact closure only on a tuned or degenerate locus
- unresolved extra low-energy observer sectors
- a missing local curvature-generating law

If any of these occur, the line must be labeled honestly as:

- ansatz
- benchmark-preserving interpretation
- partial support result
- or `Not Derived On Current Line`

## Gate-By-Gate Checklist

Mark every gate explicitly before opening the line.

- [ ] Gate 1: the new ingredient is explicit, mathematical, and not a same-output relabeling
- [ ] Gate 2: a local observer map is written down with a clear domain, codomain, and locality rule
- [ ] Gate 3: the operative metric is generated rather than merely assumed or inherited
- [ ] Gate 4: Einsteinian observer dynamics arise from the same candidate structure rather than being inserted externally
- [ ] Gate 5: extra low-energy sectors are eliminated, screened, or otherwise controlled by a stated mechanism so the benchmark claim boundary remains intact
- [ ] Gate 6: null cones, proper time, and redshift arise from that same operative metric
- [ ] Gate 7: no genuine observer-side remainder survives at benchmark scope
- [ ] Gate 8: the line states in advance what counts as `Derived`, `Not Derived On Current Line`, and `Suspended Pending New Law`
- [ ] Gate 9: the proposal explains why it is not another deeper same-output relay below the frozen bridge object
- [ ] Gate 10: the proposal preserves the benchmark exact-GR boundary unless it is explicitly being opened as a different project

If any box cannot be checked honestly, the default recommendation is `No-Go`.

## Current Recommendation

`No-Go` on opening any new derivational line unless a candidate can complete this charter concretely before manuscript promotion.

At current repository scope, no recorded same-output continuation below the frozen primitive-reservoir bridge object qualifies as a new line merely by going deeper.
The minimum admissible restart would be a proposal that writes down an explicit observer-localizing law, a concrete local map from substrate data to observer fields, a generated operative metric rule, a controlled low-energy mode story, and a bounded verdict condition.

## Conclusion

The purpose of this charter is not to forbid future derivational work.
Its purpose is to force future derivational work to become scientifically legible before it is promoted.

The benchmark package may stand even if no new line ever passes this screen.
If a future line does pass it, that line should begin as a bounded, pre-registered test rather than as another open-ended descent.
