## Already Derived?

No.

The repo’s own front door says that the effective gravitational law is adopted exactly as GR and that first-principles substrate derivation remains open. The derivation-gates note says successful derivation requires explicit substrate recovery of Einsteinian observer dynamics, one operative metric with universal matter coupling, ordinary GR gauge/two-tensor content, exact null/proper-time/redshift recovery, and no genuine observer-side remainder. The main-line frontier theorems repeatedly preserve the same benchmark package and move the unexplained layer deeper; they do not present the Einsteinian observer package as finally derived from explicit substrate microphysics.

## Impossible or Merely Unfinished?

Merely unfinished on present evidence.

- The repo contains real negative results, including screened higher-derivative branches and the minimal stable-sign no-go note.
- Those negative results are branch-local or package-local. Several notes explicitly say they are not universal impossibility theorems.
- The upstream compression theorem is also line-specific, not a universal no-go against every possible substrate derivation.
- No active manuscript currently proves that the primitive-reservoir route cannot in principle recover the benchmark one-metric observer package.

## Agent-Behavior Audit

The current recursion is not explained by impossibility alone.

- The control docs were stale: the claim-boundary, naming, checklist, and guide surfaces still pointed to older yocto- or pre-proto-germ-level frontiers even though the file map now has a single `active_primary` row at the pre-pre-pre-proto-kernel stage.
- Recent commit history from April 3 to April 5, 2026 shows a rapid sequence of `advance frontier` commits, each promoting a new theorem/criterion pair at one more substrate layer.
- The manuscript prose is highly templated. The same pattern appears again and again: preserve the same benchmark one-metric package, state that stronger promotion language remains paused, and move the “remaining honest burden” to still more primitive substrate structure.
- The overclaiming guardrails are scientifically healthy, but by themselves they do not stop recursive depth drift once the compressed benchmark object is already known.
- The repo therefore has two simultaneous facts: a real unfinished bridge burden still exists, and the execution layer has also fallen into a recursive relay habit that keeps promoting deeper same-output packages instead of stabilizing the bridge object.

## Missing Bridge Object

The first benchmark-stable missing bridge object is not the latest `active_primary` package name.

It is the substrate law that forces the primitive observer-reduction chain yielding:

- the continuity-Hessian compressed core,
- the single operative metric,
- the universal ordinary-matter route through that same metric,
- and the benchmark one-metric observer package as a derivational output rather than as preserved package data.

In the repo’s own current local terminology, the best name for that bridge object is the primitive reversible constrained substrate dynamics normal form that forces the explicit equation / symmetry / constraint package. That object is benchmark-stable in a way the current pre-pre-pre-proto-kernel relay is not.

Why this is the right object:

- The bridge-entry theorems already derive datum origin, defect-fiber factorization, one-metric/matter output maps, and chain forcing.
- The explicit-package necessity / uniqueness theorem already says that the open task is no longer to justify the explicit package itself, but to derive or justify the primitive-dynamics normal form that forces it.
- The current relay chain keeps decomposing that burden into still deeper named packages, but none of those later names has been shown to be the benchmark-stable bridge object rather than another relay.

Operational rule:

- A new main-line manuscript should count as primary only if it derives, uniquely selects, or genuinely compresses this bridge object, or discharges one of the benchmark derivation gates in a new way.
- A manuscript that merely replaces the latest package name by an even deeper one while preserving the same compressed observer package should be classified as supporting history or exploratory side progress, not as new derivational closure.

## Decision Memo

Selected outcome:

- GR is not yet derived, and the agent has fallen into recursive depth drift.

Why this is the cleanest verdict:

- Under the repo’s own derivation standard, the project still has benchmark GR adoption rather than first-principles GR derivation.
- The repo does not currently prove that the active route is impossible; the strongest negative results are scoped to screened branches or fixed-package limits.
- The benchmark-facing burden was already compressed to a much smaller bridge object, but the main-line execution kept promoting deeper same-output relays instead of stabilizing that object as the real next target.

Practical consequence:

- The right next deliverable is not another relay manuscript by default.
- The right next deliverable is any work that makes the bridge object more explicit, more necessary, more unique, or smaller in a benchmark-recognizable way.
